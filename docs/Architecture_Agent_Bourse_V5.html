Architecture Agent Bourse — Entreprise d'agents

Document technique — Arnaud Kuntz · PEA Boursobank · Mis à jour le 02/07/2026
Remplace la roadmap V5 « plus d'agents sur la V4 ». Le virage du 25/06 change le modèle de fond.

0. Le virage du 25/06 — pourquoi ce document change

Un audit honnête de la V4 a établi un fait dur, mesuré sur les données réelles : la V4 (day-trading sur 39 actions du CAC40) a une précision de 42,9%, soit sous le hasard (50%). Prédire la direction à 1 jour via analyse technique sur données gratuites est le terrain le plus efficient qui soit, où le particulier perd contre les fonds. Empiler des moteurs (l'ancienne roadmap V5) sur un signal sans edge, c'est bâtir plus haut sur du sable.

Décision : on ne jette pas la V4, on la déclasse en poche satellite (terrain de jeu plafonné). Le cœur du patrimoine passe sur une stratégie validée et éprouvée : le Dual Momentum. Et le système est repensé comme une entreprise d'agents avec une recette qualité au centre.
L'avantage compétitif visé : pas plus d'agents, pas un apprentissage plus rapide, mais la rigueur de validation. Sur 77 études d'IA-trading analysées, 2 seulement ont une séparation train/test correcte dans le temps, et aucune n'atteint un standard de reproductibilité. Être le « poisson pilote malin » qui ne met en production que ce qui passe la recette, c'est ça la différence.
Mise à jour du 02/07 — le tournant mesure. Un audit complet a trouvé et corrigé 3 bugs critiques : stats de précision empoisonnées par des prix NaN (Yahoo renvoyait des bougies incomplètes sur les valeurs .PA, non filtrées), portefeuilles jamais persistés par le CI (les trades intraday, stop-loss et take-profit étaient perdus à chaque run — les données intraday réelles ne commencent que le 02/07), et pertes silencieuses sur push git concurrent. Stats recalculées depuis zéro : baseline propre 42,7% sur 211 signaux (ACHETER 40,8%, ÉVITER 46,4%). Décision : alertes email achat/vente suspendues 1 mois (modèle non validé), et deux agents de mesure créés — Shadow (contrefactuel des alertes) et Évaluateur (edge multi-horizons). Décision de réactivation le ~02/08 sur les faits.

1. Le modèle : une entreprise d'agents à deux poches

Le capital est coupé en deux poches étanches. Un dérapage de la poche de jeu ne peut jamais entamer le cœur.

PochePartGérée parRôle
Cœur du patrimoine~85%Agent Dual Momentum (mensuel)Le vrai moteur, base solide, risque maîtrisé. On n'y touche pas.
Satellite « bons plans »~15-20% maxAgent V4 (quotidien + intraday)Terrain de jeu, chasse aux opportunités. Risque plafonné.

Au-dessus, une gouvernance type entreprise (Arnaud = maître d'ouvrage : sous-traitants, cahier des charges, recette qualité) :

  • Les employés (les agents) font leur travail et peuvent proposer des améliorations.
  • Le manager (Agent Professeur) fait le point chaque semaine, note chacun sur sa performance réelle.
  • L'éclaireur (Agent Veille) surveille ce que font les caïds et rapporte les idées à tester.
  • La recette (Harnais de mesure) est le juge : rien ne passe en production sans réussir le backtest hors-échantillon.
  • Le patron (Arnaud) valide et décide.
Règle d'or, gravée dans le système : aucune idée ne passe en production sans réussir la recette. Y compris les idées de la Veille, y compris les barèmes du Professeur. Le juge reste indépendant : le Professeur ne se réécrit pas lui-même.

3. La stratégie du cœur — Dual Momentum hybride

Idée simple : ce qui monte depuis un moment a tendance à continuer. Chaque mois on compare World et USA sur 12 mois, on garde le meilleur s'il monte (momentum absolu positif), sinon cash. Anti-whipsaw de +3% pour éviter les allers-retours stériles. Hybride : 50% du capital reste toujours sur le World (buy & hold), 50% applique la rotation.

Vérité hors-échantillon : l'hybride ne bat pas le buy & hold en rendement (le backtest in-sample qui « écrasait » le World était une illusion d'optimisation). Son vrai gain, prouvé sur données non vues : moins de risque (chute et volatilité réduites). C'est un airbag, pas un accélérateur. Les émergents ont été testés et rejetés par la recette (ils dégradaient tout en hors-échantillon).

5. Infrastructure

  • Exécution : GitHub Actions (workflow_dispatch) déclenché par cron-job.org — 10 jobs (les crons GitHub natifs, peu fiables, restent en secours dans les yml).
  • Code : Python 3.11, yfinance, pandas/numpy. CAC40 centralisé dans marche_config.py. Heures en Europe/Paris.
  • Coût : ~5 appels Claude/jour — 1 (briefing matin) + 4 (scoring intraday, régime macro Haiku). Tous les autres agents sont 100% gratuits. Coûts tracés dans costs_log.json (complet depuis le 02/07, les coûts Opus du briefing n'étaient pas persistés avant).
  • Diffusion : email Zoho SMTP + PWA iPhone.
  • Fiabilité données (02/07) : filtrage des bougies Yahoo incomplètes (Close NaN, fréquent sur les valeurs .PA tôt le matin), garde _prix_valide() dans l'évaluation de performance (NaN est truthy en Python, il passait les gardes naïfs), email d'alerte à la place du briefing si les données du jour sont inexploitables (échec visible plutôt que chiffres faux), et data_quality_log.json (90 jours glissants) pour arbitrer la diversification des sources.
  • Persistance CI réparée (02/07) : les workflows committent désormais les portefeuilles virtuels (les trades intraday n'étaient JAMAIS persistés avant, le rééquilibrage Dual Momentum du 01/07 avait été perdu) et font git pull --rebase avant chaque push (fini les pertes silencieuses sur push concurrent).
  • Stats de précision assainies (02/07) : recalcul depuis zéro après pollution NaN et double comptage — baseline propre 42,7% sur 211 signaux. Les chiffres antérieurs (41-42% sur 250-305) n'étaient pas fiables.
  • Dette technique soldée le 25/06 : timezone, bare except, duplication CAC40.
  • Revue de code 25/06 (2 relecteurs indépendants) : 7 bugs réels corrigés, dont un qui flattait la stratégie dans la recette (le World B&H honnête est passé de 12,02% à 12,53%).
  • Contrôle de cohérence devise : le mécanisme se comporte pareil en EUR (CW8/ESE) qu'en USD. Validation USD conservée car seule à contenir un krach lent (2008).
  • Expérience parallèle : portefeuille virtuel V4 à 10 000€ (depuis le 19/06), comparé au CAC 40 chaque jour dans le briefing — test empirique sans argent réel de la valeur des signaux V4.

6. Ce qui reste — idées en réserve (chacune devra passer la recette)

Court terme — deux rendez-vous datés

  • 22/07 — bilan des 30 jours : mesuré par l'Évaluateur (edge par horizon et par bucket, sur données propres depuis le 02/07). Première piste déjà visible : le bucket 85+ ne discrimine pas (il fait pire que 75-84 sur les premières données) — le seuil d'alerte à 85 est à requestionner.
  • ~02/08 — trois décisions : (1) alertes — réactiver, prolonger ou ajuster sur le verdict Shadow ; (2) poids News/Espion — supprimer, réduire, réactiver ou tester le signe contrarien News, après retest sur un mois propre ; (3) budget par position du satellite (2000€ → 500€ ?). Grille de décision (posée le 05/07) : IC significatif et edge net de frais, sinon budget symbolique ou zéro. La littérature renforce l'exigence : le trend court terme sur instruments électroniques à petit tick est structurellement mort depuis ~2009 (Bouchaud et al., arXiv 2607.01550) — le fardeau de la preuve est du côté du satellite. Le cœur Dual Momentum mensuel (signaux lents) sort renforcé du même papier.
  • Garde-fou méta du Professeur : actif dès 3 semaines d'historique

Moyen terme

  • Agent Dividendes PEA : calendrier des détachements CAC 40, rendements — 100% factuel, zéro modèle à valider (quick win)
  • Agent Patrimoine global : raisonner patrimoine entier (PEA réel, seuil fiscal 5 ans, DCA, épargne, projets) — nécessite une session dédiée avec les données réelles
  • Crypto Dual Momentum (BTC/ETH/SOL, refuge stablecoin) : l'airbag momentum vaut le plus sur cette classe d'actifs — à valider via le harnais
  • Agent Data Quality complet : distinguer « donnée manquante » de « vrai neutre », bloquer le signal si données pourries (premier étage posé le 02/07 avec data_quality_log)
  • Diversification des sources de données (sortir de la dépendance Yahoo unique) — data_quality_log fournira les faits pour arbitrer
  • Veille → Professeur → recette : boucle d'intégration automatique des idées validées

Réserve (anciens moteurs V5, seulement si la recette le justifie)

  • Feature Engine, Score Engine 4 sous-scores, Risk Engine — utiles uniquement si on prouve d'abord un edge à améliorer
  • Agent Macro+ complet (régimes avancés, HMM) — la détection de régime simple est déjà en prod dans le scoring V5. Agent Insider (déclarations AMF)
  • Bon de commande PWA semi-automatique — prérequis : un edge réel validé hors-échantillon, pas la précision V4 actuelle
